# Kanban App

Внутреннее Kanban-приложение (Trello-подобная доска задач) для Almaly AMC. Backend на Flask + SQLite, авторизация через Google Sheets с fallback на локальную БД.

## Возможности

- **Рабочие пространства (workspaces) и доски** — группировка досок по компаниям/проектам, свой цвет и фон у каждой доски.
- **Колонки и карточки** — drag-and-drop, изменение порядка, перемещение карточки в другую колонку или на другую доску, дублирование карточек/колонок/досок, архив с восстановлением.
- **Карточки**: описание с Markdown-разметкой, метки (label + цвет), срок выполнения, чек-листы (с исполнителями и сроками пунктов), комментарии, вложения (до 16 МБ), участники (members), связь с другой доской.
- **Профиль пользователя** — имя, цвет/фото аватара, смена пароля (доступен со списка досок и с самой доски).
- **Управление пользователями** — создание/редактирование/удаление пользователей и выдача доступа к доскам (для админа).
- **Экспорт в CSV** — по доске или по рабочему пространству.
- **Импорт из Trello** — одна доска (`import_trello.py`) или целый workspace из папки JSON (`import_trello_workspace.py`).
- **Поиск** по карточкам (`/api/search`).
- **Права доступа** — админ видит все доски, обычный пользователь — только назначенные (`board_access` / столбец «Доски» в Google Sheets).
- **Мобильная адаптация** — панель фильтров и адаптивная сетка колонок.
- **Health-check** — `GET /health` для мониторинга.

## Стек

- Python 3.11, Flask
- SQLite (файл `kanban.db`; схема создаётся в `init_db()`, дальнейшие изменения — в `migrate_db()` при каждом старте приложения)
- Google Sheets API (`gspread`) — опционально, для управления пользователями из таблицы
- Waitress — production WSGI-сервер (Windows)

## Структура проекта

```
app.py                       # роуты и бизнес-логика (Flask)
models.py                    # схема SQLite, init_db(), сид тестовых данных
sheets.py                    # интеграция с Google Sheets (пользователи и права)
import_trello.py             # импорт одной экспортированной доски Trello в существующую доску
import_trello_workspace.py   # импорт папки JSON-досок Trello в workspace (создаёт доски)
serve.py                     # запуск через waitress (production)
requirements.txt
.env.example                 # шаблон переменных окружения
DEPLOY_ROADMAP.md            # чек-лист и инструкции по развёртыванию на сервере
templates/                   # HTML (Jinja2)
static/                      # CSS, JS, загруженные фоны досок и аватары
uploads/                     # вложения карточек (по board_id)
kanban.db                    # база данных SQLite (создаётся автоматически)
google_credentials.json      # сервисный аккаунт Google (секрет, не в git)
```

## Установка (локальный запуск)

```
pip install -r requirements.txt
```

Скопировать `.env.example` в `.env` и задать значения:

```
SECRET_KEY=<случайная строка 32+ символов>
GOOGLE_CREDENTIALS_PATH=google_credentials.json
USERS_SHEET_ID=<ID таблицы, опционально>
```

Сгенерировать `SECRET_KEY`:
```
python -c "import secrets; print(secrets.token_hex(32))"
```

Запуск для разработки:
```
python app.py
```
Приложение поднимется на `http://localhost:5001`. При первом запуске БД и тестовые данные создаются автоматически.

Запуск в режиме, близком к production (без debug, через waitress):
```
python serve.py
```

## Авторизация

Приложение поддерживает два режима, переключается автоматически:

1. **Google Sheets** (приоритетный) — активен, если задан `USERS_SHEET_ID` и существует файл из `GOOGLE_CREDENTIALS_PATH`. Структура листа: `Email | Имя | Пароль | Роль | Доски` (роль `admin`/`user`, «Доски» — ID через запятую или `*` для всех).
2. **Локальная SQLite** (fallback) — если Google Sheets не настроен. При первом запуске создаются тестовые аккаунты:
   - `admin@almaly.kz` / `admin123` (роль admin, доступ ко всем доскам)
   - `user@almaly.kz` / `user123` (роль user, доступ к первым двум доскам)

   ⚠️ **Обязательно смените или удалите эти тестовые учётные записи перед реальной эксплуатацией.**

## Импорт из Trello

### Одна доска

Доска с указанным `--board-id` должна уже существовать в приложении:

```
python import_trello.py board-export.json --board-id 3
python import_trello.py board-export.json --board-id 3 --skip-archived
```

### Целый workspace

В Trello нет экспорта workspace целиком — экспортируйте каждую доску отдельно в JSON, положите файлы в одну папку (подпапки тоже сканируются) и запустите:

```
python import_trello_workspace.py ./trello_export --workspace "Компания А"
python import_trello_workspace.py ./trello_export --workspace "Компания А" --skip-archived
python import_trello_workspace.py ./trello_export --workspace "Компания А" --workspace-color "#0052cc" --board-color "#4361EE"
```

Скрипт найдёт или создаст workspace с указанным именем, для каждого валидного JSON создаст новую доску и импортирует содержимое. Ошибка в одном файле не откатывает уже импортированные доски.

Экспорт доски из Trello: *Меню доски → Показать меню → Ещё → Печать и экспорт → Экспорт в JSON*.

**Что импортируется:** списки → колонки, карточки (название, описание, метка, срок, архив), чек-листы, комментарии (из `actions`, до ~1000 последних действий на доску). Цвет доски при workspace-импорте берётся из фона Trello, если не задан `--board-color`.

**Что не импортируется:** вложения, участники карточек, фоны-картинки досок.

## Известные ограничения

- Пароли в Google Sheets хранятся в открытом виде — приемлемо только если доступ к таблице ограничен администраторами.
- Максимальный размер одного вложения — 16 МБ (`MAX_CONTENT_LENGTH` в `app.py`).
- При дублировании карточки/колонки/доски комментарии и вложения не копируются (как в Trello).
- Импорт из Trello не переносит вложения и участников карточек; комментарии ограничены последними ~1000 actions экспорта Trello.
